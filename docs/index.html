Systematic evaluation of native and text-based tool calling across 32 open-source LLMs from 11 providers, tested under 4 flag combinations (stream × think) with 7 tests covering core plumbing, behavioral stress, and schema complexity.

Key Findings

9 findings from 1,792 test runs across 32 models

Finding #1
No size-quality correlation
ministral-3:3b (3B params) scores 60/60. deepseek-v3.2 (671B params) scores 16/60. Model size is not a predictor of tool calling capability.
3B 60
671B 16
Finding #2
Layer independence
Native and text-based tool calling are orthogonal capabilities. Group GB (8 models) proves native competence does not imply text competence. Group GC (8 models) proves the reverse.
Finding #3
Flag agnosticism equals reliability
All 10 perfect models are completely flag-agnostic — identical behavior regardless of streaming or thinking settings. This is the strongest signal of production readiness.
Finding #4
Streaming degrades, never improves
Across 32 models and 1,792 runs, stream=true never rescued a failing test. It occasionally degraded passing tests and significantly amplified transient server errors.
Finding #5
Thinking is a wild card
think=true can help or hurt specific tests on the same model. deepseek-v3.1's FPFP/PFPF alternation: thinking fixes single-tool but breaks multi-step consistency.
Finding #6
Server stability is a measurement variable
Three models changed by >5 points between sweeps due to server behavior, not model behavior: cogito (−40T), qwen3-vl:235b (+40T→P), minimax-m2 (−7T).
Finding #7
Family consistency
Models in the same family share failure patterns: GLM text fallback broken; GPT-OSS parallel FFFF + enum hallucination; Gemma3 zero native, perfect text; Mistral boundary confusion at x=15,y=15,z=15.
Finding #8
Voxel tests are the real discriminator
25/32 models pass single/multi/parallel on baseline, but only 10/32 pass all voxel sub-checks. Schema complexity reveals capability differences invisible to simpler tests.
Finding #9
Measurement reproducibility is high
14/32 models are fully deterministic across independent sweeps. Consistent failure signatures (parallel FFFF, v_int FFFF, GLM text FFFF) reproduce identically.

Test Suite

7 tests across 3 tiers of complexity

CORE Core Plumbing (3 tests)

single_tool — 1 tool (get_weather). “What is the current weather in Tokyo?” Validates basic tool invocation.

multi_tool — 3 tools. “What's the weather like in Paris?” Validates correct selection from a set.

parallel_calls — 3 tools. “Weather in London and population of London?” Validates multiple tool calls in a single response.

STRESS Behavioral Stress (2 tests)

multi_step — 1 tool, 5 turns. Sequential weather prompts for Tokyo, Paris, New York, London, Sydney. Validates format consistency across conversation turns.

tool_count_scaling — 1→15 tools. “Current weather in Tokyo?” at scale 1, 3, 5, 8, 10, 15. Validates selection under proliferation.

SCHEMA Schema Complexity (2 tests × 5 sub-checks)

voxel_tools (native) and voxel_tools_text (text fallback) — 4 tools (placeBlock, removeBlock, getWorldState, done) with complex parameter constraints.
